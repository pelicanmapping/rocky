virtual TileKey bestAvailableTileKey(const TileKey &key) const

Given a TileKey, returns a TileKey representing the best potentially available data at that location.

This will return either: (a) the input key, meaning data is available for that exact key (b) an ancestor key, meaning a lower resolution tile is available (c) TileKey::INVALID, meaning data is NOT available for the input key’s location. Properties affecting the result include dataExtents, minLevel, maxLevel, minResolution, and maxResolution. Note: there is never a guarantee that data will be available anywhere, especially for network resources. This method is a best guess based on the information available to the layer.

Parameters:

key – Tile key to check

Returns:

Best available tile key
