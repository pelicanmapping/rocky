Entity Component System

Note

This guide is under development. For now, please refer to the API Reference for detailed API information.

Overview

Rocky uses an Entity Component System (ECS) architecture for creating map annotations and dynamic objects. The ECS is based on the EnTT library.

Key Concepts

Entities

Entities are lightweight identifiers (handles) that represent objects in your scene. They have no data themselves, but serve as containers for components.

Components

Components are data structures that define specific aspects of an entity:

  • rocky::ecs::Transform - Position, orientation, scale

  • rocky::ecs::Mesh - 3D geometry

  • rocky::ecs::Line - Line geometry

  • rocky::ecs::Point - Point features

  • rocky::ecs::Label - Text labels

Registry

The rocky::ecs::Registry manages all entities and their components.

Basic Usage

Creating an Entity

#include <rocky/ecs/Registry.h>
using namespace rocky::ecs;

Registry registry;
auto entity = registry.create();

Adding Components

auto& transform = registry.emplace<Transform>(entity);
transform.position = GeoPoint(SRS::WGS84, -122.419, 37.775, 0.0);

auto& label = registry.emplace<Label>(entity);
label.text = "Hello, World!";
