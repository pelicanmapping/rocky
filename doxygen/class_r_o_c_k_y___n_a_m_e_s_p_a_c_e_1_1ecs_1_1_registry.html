Detailed Description

Wraps the ECS registry with a read-write lock for thread safety.

Take an exclusive (write) lock when calling entt::registry methods that alter the database, like:

  • create, destroy, emplace, remove

Take a shared (read) lock when calling entt::registry methods like:

  • get, view
  • and when updating components in place

Read ROCKY_NAMESPACE::ecs::Registry::read ( ) const

Returns a reference to a read-locked EnTT registry.

A read-lock is appropriate for get(), view(), and in-place updates to existing components. The read-lock is scoped and will automatically release at the closing of the usage scope.

usage: auto [lock, registry] = ecs_registry.read();

Returns
A tuple including a scoped shared lock and a reference to the underlying registry
